Features

Everything you need, nothing you don't

Pure software solution. No IP KVM switch, no USB hub, no dongles required.

🖱️

Edge Switching

Move your mouse past the right or left edge of a screen and input instantly jumps to the neighbouring laptop. Feels like a multi-monitor setup.

⌨️

Hotkey Control

Ctrl+Alt+→ and Ctrl+Alt+← force-switch screens at any time — even when the mouse is in the middle of a screen.

📡

mDNS Auto-Discovery

Clients find the server automatically on the LAN via Zeroconf/mDNS. Use --discover and skip manual IP entry entirely.

🔒

Clean Key Release

When switching screens, a Release All packet is sent to the previous laptop so no keys remain stuck. No ghost keypresses, ever.

🔄

Auto-Reconnect

Client reconnects automatically with exponential backoff (1 s → 30 s max) if the server restarts or the network hiccups.

🛠️

Single Binary

Build once, copy anywhere. One netinput-server and one netinput-client binary. No runtime dependencies beyond the kernel.

Quick Start

Up and running in 3 steps

Run the installer once on every laptop, then start the server and clients.

1
Install on every laptop (server + all clients)

The script installs Go, system libraries, udev rules, adds your user to the input group, and builds both binaries.

# Clone the repo on each laptop
git clone https://github.com/stoxar06/NetInput_Share-GO.git
cd NetInput_Share-GO

# Run installer (once per machine)
chmod +x install.sh && ./install.sh

# Log out and back in so input group takes effect, then verify:
ls -la /dev/uinput   # should show input group with rw
2
Start the server (laptop with physical KB + mouse)

The server auto-detects your keyboard and mouse. Pass --keyboard and --mouse flags to pick specific devices.

# Start with default config.json
./netinput-server

# Or specify devices manually
./netinput-server --keyboard /dev/input/event3 --mouse /dev/input/event5

# List detected devices if unsure
ls /dev/input/by-id/
3
Start clients on each remote laptop

Each client needs a unique --id (1, 2, or 3). Use --discover to find the server automatically, or pass the IP directly.

# Option A — auto-discover server on LAN
./netinput-client --discover --id 1    # Laptop 2 (Screen 1)
./netinput-client --discover --id 2    # Laptop 3 (Screen 2)
./netinput-client --discover --id 3    # Laptop 4 (Screen 3)

# Option B — specify server IP
./netinput-client --server 192.168.1.100 --id 1

udev rules (auto-applied by install.sh)

# /etc/udev/rules.d/99-netinput.rules
KERNEL=="uinput",  GROUP="input", MODE="0660"
KERNEL=="event*",  GROUP="input", MODE="0660"

These rules allow members of the input group to use evdev and uinput without running as root.
